# Симуляция инвестиционных и торговых стратегий на рынке криптовалют

Этот проект содержит два скрипта на языке Python, предназначенных для исторического анализа эффективности различных подходов к инвестированию и трейдингу на криптовалютном рынке. Анализ проводится по четырем популярным активам: `BTC-USD`, `ETH-USD`, `SOL-USD` и `BNB-USD`.

---

## Требования к окружению

Для работы скриптов требуются стандартные библиотеки Python для обработки данных, загрузки котировок и построения графиков. Установите их с помощью `pip`:

```bash
pip install yfinance pandas matplotlib
```

## Описание файлов проекта

### 1. Активный трейдинг (`crypto-algs.py`)
Скрипт предназначен для тестирования классических алгоритмов автоматической торговли на данных за последний месяц. 

* **Функционал**:
  * Скачивает 4-часовые исторические данные.
  * Рассчитывает технические индикаторы:
    * Простые скользящие средние: быстрый (`SMA_10` — средняя за 10 периодов) и медленный (`SMA_30` — средняя за 30 периодов).
    * Индекс относительной силы (`RSI`) с периодом 14.
  * Симулирует торговлю с начальным капиталом **$1000** по одной из двух стратегий на выбор пользователя:
    * **SMA**: Покупка при пересечении быстрой линии выше медленной, продажа — при пересечении вниз.
    * **RSI**: Покупка при входе в зону перепроданности (ниже 30), продажа — при входе в зону перекупленности (выше 70).
  * **Визуализация**: Строит индивидуальные графики цен для каждого актива с нанесенными линиями SMA и стрелками совершенных сделок (покупка/продажа).

* **Запуск**:
  ```bash
  python trading_backtest.py
  ```
  *(После запуска скрипт попросит ввести в консоль название стратегии: `SMA` или `RSI`)*.

### 2. Пассивные долгосрочные инвестиции (`crypto-long.py`)
Скрипт моделирует и сравнивает три классических пассивных подхода к накоплению активов на **годовом интервале** (с 1 мая 2024 года по 1 мая 2025 года) с общим бюджетом в **$1000** на каждый актив.

* **Тестируемые стратегии**:
  * **HODL (Купи и держи)**: Единовременная покупка актива в первый день на всю сумму бюджета.
  * **DCA (Усреднение стоимости)**: Бюджет делится на 4 равные покупки (по $250), которые совершаются через равные промежутки времени в течение года.
  * **Staking (Стейкинг)**: Покупка по стратегии HODL, но с условным начислением 1% годовой доходности к объёму удержанных монет.
* **Визуализация**: Выводит текстовый отчет по доходности каждой стратегии в консоль и строит сетку графиков 2х2 с динамикой цен активов за выбранный год.

* **Запуск**:
  ```bash
  python investment_simulation.py
  ```
